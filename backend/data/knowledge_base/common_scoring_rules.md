# 政企软件项目常见评分规则

## 1. 评分结构

政企软件信息化项目评标通常采用综合评分法，总分100分，一般分为：

- 技术部分：50-70分（多数项目技术分占比最高）
- 商务部分：15-30分
- 价格部分：10-30分

## 2. 技术方案高分写法

### 2.1 总体架构

- 采用分层架构（展示层/业务层/数据层/基础设施层）
- 体现微服务或模块化设计思路
- 画架构图，标注技术选型
- 强调高可用、可扩展、可维护

### 2.2 功能模块设计

- 每个功能模块按"需求理解→设计思路→技术实现→亮点说明"展开
- 对照评分项逐条响应
- 用流程图、时序图、ER图辅助说明

### 2.3 安全体系

- 网络安全、应用安全、数据安全三层设计
- 引用等保标准条款
- 体现国产密码算法支持

### 2.4 实施方案

- WBS分解到周
- 关键里程碑和交付物
- 风险预案和应急措施

## 3. 商务得分项

### 3.1 业绩

- 按评分要求准备合同关键页（首页、金额页、签字页、验收页）
- 中标通知书和验收报告配套

### 3.2 资质认证

- CMMI等级对应得分
- ISO体系认证（27001、20000、9001）
- ITSS运维资质

### 3.3 团队资质

- 项目经理PMP/信息系统项目管理师
- 技术负责人高级职称/架构师认证
- 核心成员相关认证

## 4. 价格分规则

### 4.1 低价优先法

价格得分 = (最低有效报价 / 本家报价) × 价格分满分

### 4.2 综合评估法

价格得分 = 价格分满分 - 偏差率 × 扣分系数

### 4.3 报价策略建议

- 不低于成本价
- 参考预算和同类项目中标价
- 预留价格分空间
- 不做围标串标控标
