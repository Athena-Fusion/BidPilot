# BidPilot 路演大纲

## 1. 开场（30秒）

- 一句话定位：BidPilot 是面向政企软件信息化项目的智能投标Agent
- 痛点：读标慢、漏响应、废标风险高
- 方案：多Agent协作，从招标文件到响应表的完整闭环

## 2. 行业痛点（1分钟）

- 政企信息化项目招投标流程复杂
- 一份招标文件100+页，2-4小时才能初步读完
- 资格要求、★号参数容易遗漏 → 废标
- 响应表、偏离表手动填写耗时
- 新人缺乏经验，依赖老员工指导

## 3. 解决方案（2分钟）

- 多Agent协作：10个专业Agent分工明确
- 完整业务闭环：招标解析 → 资格提取 → 评分分析 → 风险识别 → 策略生成 → 方案生成 → 商务响应 → 响应表 → 合规审查 → 报告导出
- 知识库辅助：本地知识库关键词检索
- Mock模式：无需API即可完整演示

## 4. 核心功能演示（4分钟）

- 选择"智慧园区综合管理平台建设项目"
- 一键分析，展示Agent流程
- 展示招标摘要、资格要求、评分规则
- 重点展示风险清单：5项高风险
- 展示投标策略和材料清单
- 展示技术方案初稿和响应表
- 展示合规审查和免责声明
- 导出Markdown报告

## 5. 技术架构（1分钟）

- 后端：FastAPI + Python + Pydantic
- 前端：React + Vite + TypeScript + TailwindCSS
- Agent架构：BaseAgent + LLMClient + 统一上下文
- 知识库：本地Markdown + 关键词检索
- 模型抽象：Mock/OpenAI Compatible/自定义模型

## 6. 量化价值（30秒）

- 读标：2-4小时 → 10-20分钟
- 资格整理：1小时 → 5分钟
- 评分分析：1-2小时 → 10分钟
- 响应表：半天 → 10-15分钟

## 7. 扩展路线（30秒）

- 真实模型接入
- 向量RAG知识库
- Word/PDF标书导出
- 企业资质材料库
- 多项目历史记录
- 昇腾NPU适配

## 8. 总结（30秒）

- 垂直场景：政企软件信息化投标
- 完整闭环：从招标文件到响应表
- 多Agent协作：10个Agent有trace可解释
- 合规表达：辅助审查，需人工复核
- 开箱可演示：Mock模式无需API
