# BidPilot Demo 演示脚本

## 演示前准备

1. 确保后端已启动：`cd backend && uvicorn main:app --reload --port 8000`
2. 确保前端已启动：`cd frontend && npm run dev`
3. 确认 `MOCK_MODE=true`（默认）
4. 浏览器打开 http://localhost:5173

## 演示流程（约10分钟）

### 第1步：工作台介绍（1分钟）

- 打开 Dashboard 页面
- 介绍 BidPilot 定位：政企软件信息化投标智能Agent
- 展示量化价值：读标4小时→20分钟、响应表半天→15分钟
- 展示3个示例招标项目

### 第2步：选择招标文件（1分钟）

- 点击"智慧园区综合管理平台建设项目"的"一键分析"
- 进入招标分析页面
- 说明：480万预算、90天建设周期、技术60分+商务20分+价格20分

### 第3步：一键分析（2分钟）

- 点击"一键分析"按钮
- 展示 Agent 执行流程：10个Agent依次完成
  - 招标解析 → 资格提取 → 评分分析 → 风险识别 → 策略生成 → 方案生成 → 商务响应 → 响应表生成 → 合规审查 → 报告生成
- 强调：全部Agent有trace，可解释

### 第4步：招标摘要与资格要求（1分钟）

- 展示基本信息：项目名称、采购人、预算、截止时间
- 展示资格要求：带风险标签
  - 🔴高风险：CMMI3级认证、类似项目业绩
  - 🟡中风险：项目经理PMP、ISO 27001

### 第5步：评分规则（1分钟）

- 展示评分结构：技术60分+商务20分+价格20分
- 展示高权重项（★标记）：
  - ★设备接入管理方案（10分）
  - ★国产化适配方案（10分）
  - ★数据安全方案（8分）

### 第6步：风险审查（1分钟）

- 切换到"风险审查"页面
- 展示风险清单：高风险5项
  - 报价超预算废标
  - 保证金未交纳废标
  - 签字盖章不符合废标
  - ★号参数未响应废标
  - 资格条件不满足废标
- 说明：覆盖报价、资质、签章、保证金、★号参数等常见废标风险

### 第7步：投标策略（1分钟）

- 切换到"投标策略"页面
- 展示投标建议：建议参与
- 展示得分策略：重点覆盖3个10分项和★号参数
- 展示材料清单：7项待准备材料
- 强调：报价策略合规，不含围标串标建议

### 第8步：标书生成（1分钟）

- 切换到"标书生成"页面
- 展示技术方案初稿：13个章节
- 展示商务响应表
- 展示技术响应表：★号参数逐条完全响应
- 展示报告预览

### 第9步：合规审查（1分钟）

- 切换到"合规审查"页面
- 展示总体状态和通过率
- 展示致命问题：4项高风险
- 展示待人工确认项：5项
- 展示免责声明

### 第10步：导出与总结（1分钟）

- 点击"导出Markdown"
- 展示导出的报告文件
- 总结量化价值
- 说明扩展方向

## 重点强调

1. **业务闭环**：从招标文件到响应表、偏离表、合规审查的完整链路
2. **多Agent架构**：10个专业Agent分工协作，有trace可解释
3. **废标风险覆盖**：报价/资质/签章/保证金/★号参数
4. **合规表达**：所有结论标注"辅助审查，需人工复核"
5. **Mock模式**：无需任何API即可完整演示

## 答辩要点

- 行业痛点：读标慢、漏响应、废标风险高
- 技术创新：多Agent协作 + 知识库检索 + 结构化输出
- 场景落地：政企软件信息化投标垂直场景
- 量化价值：基于模拟案例估算的时间节省
- 扩展路线：真实模型、向量RAG、Word导出、企业资料库
